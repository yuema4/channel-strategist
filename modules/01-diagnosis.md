# 模块1：诊断

> 目的：看清现状。输入账号信息，输出商业模式体检报告。

---

## 工作流

```
[入口] 用户说"诊断我的账号""我账号怎么样""帮我看看商业模式"
   ↓
[Step 1.0] 收集信息（5个必填项）
   ↓
[Step 1.1] 七项检验（逐项打分，每项停在对话中等用户回应）
   ↓
[Step 1.2] 博主分析（按需，用户指定博主时）
   ↓
[输出] 诊断报告（7项分数 + 最弱3项 + 改进优先级）
```

---

## Step 1.0：收集信息

### 触发

用户说以下任一关键词，进入诊断：
- "诊断我的账号""我账号怎么样""帮我看看商业模式"
- "评估我的账号""我值不值得做""我适不适合做这个"

### AI第一句话

> "好。我先了解你的情况——告诉我你现在在做什么、怎么赚钱的、卖给谁。"

### 必须拿到以下5项信息（缺一项就追问）

| # | 信息 | 追问方式 |
|:---|:---|:---|
| 1 | **平台+粉丝数** | "你在哪个平台？多少粉？" |
| 2 | **主品类** | "你主要发什么内容？一句话描述你的频道" |
| 3 | **已变现** | "赚过钱吗？怎么赚的？接广告还是卖东西？大概多少？" |
| 4 | **产品** | "你有自己的产品吗？课程/工具/模板/咨询？把付款链接发给我看看？"（没有就说没有）|
| 5 | **目标** | "18个月后，你希望这个账号达到什么状态？" |

### 信息不完整时的处理

- 用户说"我不确定" → 追问"那你现在最确定的是什么？先说那个"
- 用户说"没有变现" → 追问"是因为没想过变现，还是想过但没跑通？"
- 用户说"没有产品" → 追问"你想过做什么产品吗？还是只想靠广告？"

### 语言陷阱检测（简化版）

在用户回答过程中，如果出现以下模糊词，**立刻停下来追问**：

| 模糊词 | 追问方式 |
|:---|:---|
| "适合" | "你觉得什么叫'适合'？你用什么标准判断适不适合？" |
| "好的" / "高级" | "你说的'好'具体指什么？能举一个你认为是'好'的例子吗？" |
| "赛道" | "你说的赛道具体指什么？你是在找内容方向还是在找变现模式？" |
| "流量" | "你要的是粉丝数还是收入？这两件事不是一回事。" |

**原则**：词没定义清楚之前，不往下走。不是刁难用户，是帮助用户想清楚自己到底在问什么。

## Step 1.1：七项检验

逐项检验，**每做完一项停下来把结论告诉用户，等用户回应后再进入下一项**。不要一次性跑完7项。

---

### 检验1：客户检验（0-10分）

> "谁在消费你的内容？你能描述清楚吗？"

**AI问用户**：
- "你的粉丝是什么人？用你的话描述一下——他们大概什么年龄、做什么工作、为什么关注你？"

**AI自己判断**（结合用户回答 + 已知数据）：
- 粉丝画像是否精确到可识别人群？（不是"年轻女性""都市白领"）
- 粉丝来的原因和用户想变现的方向是否一致？
- 有没有付费用户？付费用户和免费粉丝是同一群人吗？

**评分标准**：

| 分数 | 标准 |
|:---|:---|
| 8-10 | 能精确描述画像（年龄/职业/需求/付费意愿），且有数据支撑 |
| 5-7 | 大概知道但不够精确，或画像和变现方向不完全匹配 |
| 0-4 | 不知道谁在看，或描述泛化（"年轻女性""喜欢数码的人"）|

**红灯信号**：
- "我的粉丝什么人都有一点" → 客户画像不清晰
- "我也不知道他们为什么关注我" → 内容和客户脱节
- 粉丝要的和你想卖的不是一回事 → 客户-产品不匹配

---

### 检验2：产品检验（0-10分）

> "你有没有产品/服务？是什么？能说出它的颜色吗？"

**AI问用户**：
- "把你的付款链接发给我看看？"（能发出来 = 有产品）
- "你的产品是什么颜色的？"（说不出颜色 = 还没进入市场）

**AI自己判断**：
- 有没有自有产品？（课程/工具/模板/咨询）
- 如果没有自有产品，靠什么赚钱？（纯广告 = 没有产品）
- 广告位本身算产品吗？（算，但那是流量产品，不是IP产品）

**评分标准**：

| 分数 | 标准 |
|:---|:---|
| 8-10 | 有自有产品（课程/工具/模板/咨询），已有收入 |
| 5-7 | 有第三方变现（广告），但无自有产品 |
| 0-4 | 没有产品，或"想做但还没做" |

**红灯信号**：
- "我打算以后做XX" → 现在就是没有产品
- "我在接广告" → 广告是流量变现，不是产品变现
- "我的内容本身就是产品" → 视频不是产品，视频里教的方法论包装成课程才是产品

---

### 检验3：触达检验（0-10分）

> "用户通过什么路径找到你？这个路径稳定吗？"

**AI问用户**：
- "你的粉丝主要从哪来的？搜索你的名字？平台推荐？还是别人转发？"
- "如果你明天不发新内容，还有人能看到你的老内容吗？"

**AI自己判断**：
- 流量来源是单一还是多元？
- 有没有搜索流量？（搜索流量 = 用户主动找，最值钱）
- 有没有社交传播？（转发/截图分享 = 免费增长）

**评分标准**：

| 分数 | 标准 |
|:---|:---|
| 8-10 | 多平台+多入口（搜索/推荐/社交转发），老内容有持续流量 |
| 5-7 | 单平台但流量稳定，主要靠推荐 |
| 0-4 | 依赖单一流量入口，随时可能断；或不发新内容就没流量 |

**红灯信号**：
- "主要靠平台推荐" → 算法一变就没流量
- "不发新内容就没有浏览量" → 内容没有长尾价值
- "只在抖音/小红书一个平台" → 单平台风险

---

### 检验4：交付检验（0-10分）

> "你的产品/内容以什么形式给到用户？交付过程顺畅吗？"

**AI问用户**：
- "用户付钱后，他怎么拿到你的东西？是自动的还是要你手动操作？"
- "如果你一周不看手机，用户付了钱能自己收到产品吗？"

**AI自己判断**：
- 有没有标准化交付流程？
- 交付是否需要用户的时间参与？（需要 = 不能规模化）
- 广告主怎么投放？（品牌方自己交付 vs 你帮品牌方做内容）

**评分标准**：

| 分数 | 标准 |
|:---|:---|
| 8-10 | 有标准化交付流程（自动发送/模板下载/录播课程）|
| 5-7 | 有内容但无产品交付；或广告交付成熟 |
| 0-4 | 内容和产品都没有交付体系 |

**红灯信号**：
- "客户付了钱我手动发文件" → 不能规模化
- "我需要跟每个客户1v1沟通" → 你的时间就是天花板
- 广告方面：品牌方主动找 vs 你主动找 → 被动接单不是稳定的交付

---

### 检验5：收入检验（0-10分）

> "收入模型稳定吗？客单价合理吗？"

**AI问用户**：
- "你现在的收入来源有几种？各占多少？"
- "最稳定的那个收入，一个月能有多少？能持续多久？"
- "你的引流款和利润款价格差几倍？"

**AI自己判断**：
- 收入来源是否多元？（≥2个 = 健康）
- 客单价是否合理？（内容创作者的引流款 × 利润款价格差最好≥5倍）
- 收入是否可持续？（一次性 vs 复购型）

**评分标准**：

| 分数 | 标准 |
|:---|:---|
| 8-10 | 有定价体系+多元收入+稳定增长趋势 |
| 5-7 | 有收入但来源单一或定价/频次不稳定 |
| 0-4 | 无收入、收入不可持续、或定价完全随意 |

**红灯信号**：
- "我只靠广告" → 单一收入源
- "我接一次广告X元"但没有复投 → 不是稳定收入
- 引流款和利润款是同一个价格 → 定价即产品，你没有产品分层

---

### 检验6：留存检验（0-10分）

> "用户消费完了还会回来吗？有没有复购设计？"

**AI问用户**：
- "你的粉丝里，有人买过你超过一次东西吗？是什么？"
- "如果有人今天买了你的东西，6个月后TA还可能买你什么？"
- "广告主投过一次的，有几个投了第二次？"

**AI自己判断**：
- 有没有复购链路（课程体系/社群/模板升级）？
- 广告主有没有复投？（复投 = 客户留存）
- 粉丝粘性如何？（评论区有没有"老粉"发言）

**评分标准**：

| 分数 | 标准 |
|:---|:---|
| 8-10 | 有复购链路（课程体系/社群/订阅制/模板升级）且已有复购数据 |
| 5-7 | 有复购潜力但没主动设计 |
| 0-4 | 一次性消费，无复购设计，广告主不复投 |

**红灯信号**：
- "广告主投了一次就没后续了" → 你的内容对品牌方的留存价值不够
- "粉丝买了一次模板就不来了" → 没有升级/系列化产品
- 没有社群/订阅/复购链路 → 每次都是新客户，获客成本无限高

---

### 检验7：增长检验（0-10分）

> "你有没有杠杆？换一个人来做你的事，能做成吗？"

**AI问用户**：
- "你现在做的事，如果找一个人来替你，他能做吗？"
- "你发一条内容涨的粉，跟你一个月前发一条涨的粉，数量差多少？"
- "你做的事情里，有没有什么是不需要你亲自做的？"

**AI自己判断7个成长层级**：

| 层 | 描述 | 核心任务 |
|:---|:---|:---|
| 1 | 有人需要这个内容 | 验证需求存在 |
| 2 | 有人愿意关注/收藏 | 完成第一条爆款 |
| 3 | 有稳定的涨粉 | 找到可重复的内容模式 |
| 4 | 有品牌方找上门 | 验证商业价值 |
| 5 | 品牌方复投 + 粉丝付费 | 建立双轨变现 |
| 6 | 多产品协同（广告+模板+课程+工具）| 建立产品矩阵 |
| 7 | 行业标准制定者 | 定义规则 |

**不能跳层。** 如果用户在第3层想着第7层的事，直接指出。

**评分标准**：

| 分数 | 标准 |
|:---|:---|
| 8-10 | 有杠杆（产品/团队/IP/技术），增长持续，在第5层以上 |
| 5-7 | 正在建杠杆，增长可期，在第3-4层 |
| 0-4 | 无杠杆，增长靠运气，在第1-2层 |

**红灯信号**：
- "所有事都是我一个人做" → 没有团队，你就是天花板
- "我的增长完全靠平台推荐" → 不是你在增长，是平台在赏饭
- 在第2层想着第5层的产品矩阵 → 跳层，先验证第3层再说

## Step 1.2：博主三层蒸馏（按需，内置）

### 什么时候触发

用户说以下任一关键词，进入博主分析：
- "分析一下这个博主""拆解博主""蒸馏博主"
- "我想学XX博主的风格""XX博主为什么这么火"
- 用户提供了博主主页链接

### 方式

AI 引导用户观察博主内容，按三层蒸馏框架逐一分析。不需外部工具，不需 API。

### 博主分析的前置信息

AI 第一句话：
> "好。告诉我这个博主叫什么，在哪个平台，大概多少粉。你最近看过 TA 哪几条内容让你印象最深？"

必须拿到：
1. 博主名 + 平台 + 大概粉丝数
2. 用户印象最深的 2-3 条内容（标题 + 为什么觉得好）
3. TA 大概发什么内容，一句话描述

---

### 三层蒸馏

逐层分析，**每做完一层停下来把结论告诉用户，等用户回应后再进入下一层**。

---

#### 第一层：认知层 — TA怎么想

> "这个博主的底层信念是什么？TA 看世界的方式和别人有什么不同？"

**1. 核心信念提取（3-5 条）**

AI 引导用户观察：
- TA 在不同内容里反复出现的观点是什么？
- TA 最常说的判断句式是什么？（"XX 的关键不是 YY，是 ZZ"）
- 如果只能用一个词形容 TA 的价值观，是什么？

输出格式：
```
1. "{信念}" — 出现在《笔记A》《笔记B》中 | 验证：跨主题复现 ✅
```

**2. 观点张力**

找矛盾：TA 有没有在不同内容里表达过互相矛盾的观点？
- 矛盾不是 bug，是真实的标志。完全自洽 = 太假了。
- 格式：观点A（出处）vs 观点B（出处）+ 如何理解这个矛盾

**3. 价值立场**

- TA 的内容底色是什么？（用一句话说清 TA 在"捍卫"什么）
- TA 的受众为什么要信 TA 而不是别人？

**4. 与读者的关系**

- TA 怎么称呼读者？TA 和读者的关系是导师/朋友/学姐/同行？
- TA 在评论区怎么跟读者互动？

---

#### 第二层：策略层 — TA怎么运营

> "TA 是怎么规划内容的？有什么运营规律？"

**1. 内容系列**

- TA 有没有固定系列？（从内容标题/标签规律判断）
- 系列之间是什么关系？（独立并行？递进？交替？）
- 非系列内容的占比和定位？

**2. 发布节奏**

- TA 多久发一次？（高频/中频/低频）
- 有没有固定的发布时间段？
- 系列内容和单篇内容的比例？

**3. 蹭热点方式**

- TA 蹭热点吗？频率如何？
- 热点内容和常规内容的数据对比？（哪个表现更好）

**4. 标签策略**

- TA 每篇固定带哪几个标签？
- 有没有区别"大标签"和"小标签"的搭配规律？

**5. 运营决策准则（3-5 条）**

从 TA 的行为推断运营习惯：
```
If {可观测的条件} → Then {TA 的应对方式}
证据：《笔记名》中观察到...
```

---

#### 第三层：内容层 — TA怎么写

> "TA 的内容有什么固定公式？换一个人能写出同样的感觉吗？"

**1. 标题公式（TOP5）**

分析 TA 最近内容的标题规律：

| # | 公式名 | 使用率 | 模板 | TA 原标题示例 |
|---|--------|--------|------|-------------|
| 1 | {类型} | X% | "{模板}" | "{原标题}" |

常见类型：数字型 / 疑问型 / 感叹型 / 教程型 / 故事型 / 悬念型 / 对比型 / 列表型

**2. 开头模板（TOP3）**

TA 怎么开头？分析前 50 字：
- 类型1：{名称} — 结构 → 仿写模板
- 类型2：{名称} — 结构 → 仿写模板
- 类型3：{名称} — 结构 → 仿写模板

**3. 正文骨架**

TA 的典型正文结构是什么样的？
```
{开头}（约X字）→ {主体}（约X字）→ {收尾}（约X字）
```

**4. 情感节奏**

TA 的内容情绪走势是怎样的？
- 主导模式：低开高走 / 平稳递进 / 波浪起伏？
- TA 用什么方式制造情绪波动？（反转/自嘲/悬念/共鸣）

**5. 语言指纹**

- 高频用语 TOP5：TA 的"口头禅"级词汇
- 句式节奏：短句密集还是长句铺陈？口语化还是书面化？
- 标点/emoji 使用习惯

**6. CTA 策略**

TA 怎么引导互动？
- CTA 类型（评论/收藏/关注/转发）
- 放置位置（开头/正文中/末尾）

**7. 视觉规则**

- 视频为主还是图文为主？
- 封面有什么固定风格？
- 排版/剪辑有什么辨识度特征？

---

### 输出格式

三层蒸馏完成后，输出一份博主分析报告：

```markdown
# {博主名} 博主分析报告

## 一、一眼看清
- 平台：[小红书/B站/抖音]
- 粉丝：约X
- 定位：[一句话]
- 底层公式：[用 × 连接的核心要素]

## 二、人设拆解
- 人设标签：[3 个关键词 + 每个一句话解释]
- 粉丝为什么追 TA：[1-2 句话]
- 护城河：[为什么别人难以复制]

## 三、认知层
- 核心信念（3-5 条）
- 价值立场
- 与读者的关系

## 四、策略层
- 内容系列
- 发布节奏
- 标签策略
- 运营决策准则

## 五、内容层
- 标题公式 TOP5
- 开头模板 TOP3
- 正文骨架
- 情感节奏
- 语言指纹
- CTA 策略
- 视觉规则

## 六、对你有什么用
- ✅ 立刻可以复制的：[1-3 条]
- ⚠️ 需要适配的：[1-3 条]
- ❌ 不适合你的：[1-3 条]

## 七、一句话
{犀利总结}
```

### 快速模式

如果用户只是想快速了解一个博主（不走完整三层），用精简版：

```
# {博主名} 快速分析

定位：[一句话]
爆款规律：[TA 的爆款内容有什么共同特征]
标题公式：[最常用公式 + 示例]
变现方式：[广告/产品/课程/其他]
可学什么：[3 条具体可抄的东西]
```

## 诊断报告模板

七项检验全部完成、每项都跟用户讨论过之后，整理成报告：

```markdown
# [账号名] 频道战略诊断报告

## 一、账号概况
- 平台：...
```

## 诊断全程铁律

1. **每一步都对话**。不要闷头跑完7项。做完一项就把结论抛出来，等用户回应。
2. **消解优先**。用户问的问题如果本身不成立（模糊词/错误假设），先消解，再诊断。
3. **有判断不比没判断好**。数据不全就用已有信息判断，判断错了比不判断好。
4. **不推荐"去做市场调研"**。用户来找你就是因为不知道怎么做。你要给出判断，不是给出任务。
5. **一句话处方必须有**。诊断报告的最后，一句话告诉用户最核心的问题是什么。

## 反模式

❌ **跳过信息收集直接打分** → 信息不全，打分不准
❌ **一次性跑完7项不跟用户对话** → 诊断是对话，不是报告生成器
❌ **模糊词不追问** → "适合""赛道""高级"等词没定义清楚就继续 = 浪费双方时间
❌ **替用户跳过红灯信号** → 看到红灯要说，不是"等下次再说"

✅ **正确顺序**：收集信息 → 逐项检验（每项对话）→ 出报告 → 问用户"你对这份报告有什么不同意的地方吗？"
