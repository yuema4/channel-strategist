# 模块4：复盘

> 目的：验证假设。回答3个问题——计划跑通了吗？哪里跑偏了？需要调整什么？

---

## 工作流

```
[入口] 已执行一段时间（有数据可看）
   ↓
[3个时点]
├── 周复盘（每周）
├── 月复盘（每月）
└── 季度复盘（每3月）
   ↓
[输出] 调整方案（基于数据的下一步）
```

---

## 时点1：周复盘

### 频率
每周日（建议固定时间）

### 数据源
- 小红书/抖音/B站后台数据
- 已发布内容的赞/藏/评/转发
- 涨粉数

### 必看的4个指标

| 指标 | 怎么算 | 健康线 |
|:---|:---|:---|
| **均赞** | 总赞/总笔记 | >500（你的赛道平均）|
| **均藏** | 总藏/总笔记 | >200 |
| **涨粉/条** | 净增粉/发视频数 | >50（视阶段）|
| **互动率** | (赞+藏+评)/曝光 | >5% |

### 复盘3问
1. **这周哪条表现最好？为什么？**
   - 找到"成功因子" → 复制到下周
2. **这周哪条表现最差？为什么？**
   - 找到"失败因子" → 避免在下周重犯
3. **这周的内容是否强化了主B定位？**
   - 如果没有 → 调整下周清单

### 输出格式

```markdown
# 周复盘（YYYY/MM/DD - YYYY/MM/DD）

## 数据
- 本周发布：X条
- 总赞：X / 总藏：X / 总评：X
- 涨粉：X
- 主B占比：X%

## 表现最好
- [内容名]（X赞）
- 成功因子：[原因]

## 表现最差
- [内容名]（X赞）
- 失败因子：[原因]

## 主B定位强化？
- ✅ / ⚠️ / ❌ + 说明

## 下周调整
- 继续做：[X]
- 改做：[Y]
- 不再做：[Z]
```

## 时点2：月复盘

### 频率
每月最后一天

### 数据源
- 累计数据（30天）
- 商务进展（接了几个广告/BD了几个）
- 商业化进展（变现了多少）

### 必看的6个指标

| 指标 | 关注点 |
|:---|:---|
| 累计涨粉 | 节奏是否达标 |
| 累计变现 | 商业化是否跑通 |
| 客户结构 | 塔尖/塔身/塔底占比 |
| 内容配比 | 是否守住5:2:2:1 |
| 主B占比 | 战略主轴是否强化 |
| 对标博主数据 | TA在做什么我能否借鉴 |

### 复盘3问
1. **这个月商业化跑通了吗？**
   - 接了几个广告？变现多少？
   - 客户结构往哪边走？
2. **这个月主B定位强化了吗？**
   - 看主B占比、爆款内容、客户主动找来
3. **这个月赛道还成立吗？**
   - 是否需要重蒸馏对标博主？

### 输出格式

```markdown
# 月复盘（YYYY/MM）

## 数据
- 累计涨粉：X
- 累计变现：X
- 客户结构：塔尖X% / 塔身X% / 塔底X%
- 主B占比：X%

## 商业化
- 接广告：X个
- 主动BD：X个
- 客户主动找来：X个
- 跑通的产品：X个

## 战略进展
- 主B强化？[是/否]
- 18月节奏：处于[阶段X]，[达标/未达标]

## 调整
- 客户结构：[怎么调]
- 配比：[怎么调]
- 节奏：[怎么调]

## 下月重点
1. [P0]
2. [P1]
3. [P2]
```

## 时点3：季度复盘

### 频率
每3个月末

### 数据源
- 累计3个月数据
- 战略目标进展
- 市场变化（小红书规则、对标博主动向）

### 必看的5个问题

| # | 问题 | 输出 |
|:---|:---|:---|
| 1 | 18月节奏走到第几阶段了？| [阶段X]，是否提前/滞后 |
| 2 | 主B定位是否还成立？| 是/否/调整 |
| 3 | 客户金字塔进展？| 塔尖占比从X%→Y% |
| 4 | 商业模式7项重新打分 | 旧分→新分 |
| 5 | 是否需要重新蒸馏对标博主？| 是/否 |

### 季度复盘的"重做"动作

- **重新蒸馏对标博主**（每3-6月1次）
- **重做客户候选清单**（每3月1次）
- **重做18月节奏表**（每6月1次）
- **重做客户金字塔目标**（每6月1次）

### 输出格式

```markdown
# 季度复盘（YYYY QX）

## 18月节奏
- 计划走到：阶段X
- 实际走到：阶段Y
- 偏差：Z

## 商业模式7项（重新打分）
- 旧分：[X/70]
- 新分：[Y/70]
- 改进项：[哪3项变了]

## 客户结构
- 3个月前：塔尖X% / 塔身Y% / 塔底Z%
- 现在：塔尖X% / 塔身Y% / 塔底Z%
- 趋势：[爬升/停滞/下滑]

## 对标博主
- 蒸馏过：[A/B/C]
- 需要重新蒸馏：[A/B/C]
- 新增对标：[D]

## 战略调整
- 继续：[原计划]
- 调整：[新计划]
- 放弃：[不做的事]

## 下季度重点
1. [战略层]
2. [战术层]
3. [执行层]
```

---

## 复盘的反模式

❌ **只看数据不下结论** → 复盘必须有调整动作
❌ **只看短期不对比长期** → 周复盘看本周，月复盘看趋势
❌ **只看自己不看对标** → 季度必须看对标博主在做什么
❌ **调整过快/过慢** → 数据不足就调整 = 拍脑袋；数据超1月还调整 = 拖延

✅ **正确节奏**：周/月数据 → 季度对照对标 → 年度重定位

---

## 复盘触发器（关键词）

| 关键词 | 触发模块 |
|:---|:---|
| "周复盘" / "本周数据" | 周复盘 |
| "月复盘" / "这个月怎么样" | 月复盘 |
| "季度复盘" / "3个月了" | 季度复盘 |
| "调整" / "重做清单" | 排节奏模块（修订单）|

---

## 输出契约

每次复盘必须输出：
1. **数据快照**（4-6个关键指标）
2. **判断**（3个问题的回答）
3. **调整动作**（具体到下周/下月/下季度）

**禁止**：
- ❌ 只报告数据不给判断
- ❌ 只说"还行""不太好"无具体动作
- ❌ 跳过调整环节（复盘 = 改 + 验证）
